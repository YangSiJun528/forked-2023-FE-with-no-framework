Github 어딘가에서 누군가 스터디 한 내용을 포크해서 좀 더 정리한 레포.

그 외엔 책 저자가 [Github에 올린 코드](https://github.com/Apress/frameworkless-front-end-development)를 보거나, 서점에서 번역된 책을 보거나 했다. 


# 후기

어떤 프레임워크를 쓸 때 내부 동작 원리를 대충이라도 모르면 궁금증 때문에 집중이 안되는 사람으로써, 이런 괜찮은 자료가 있어서 빠르게 학습할 수 있었음. 

내부 구현을 보면서 현대적인 SPA(+ Redux 같은 상태관리)의 내부 동작을 어느정도 이해할 수 있었다.

2022-2023년도에 FE 찍먹하면서 있었던 궁금증이 어느정도 해소되었다.
- 컴포넌트를 따로 분리하는가? 
- 변수가 아니라 state라는게 있는가? 
- Hooks 같은 함수를 호출해야만 하지?
- url 뒤에 `/#/` 같은게 붙는건가? 
- Redux는 또 뭐지?? 

다만 실제로 이제 SPA로 사이트를 만들어낼수 있냐고 하면, 그건 아닌 것 같다. 

동작하는 코드는 만들어 낼 순 있어도 실제로 고려해야 할 부분이 매우 많을거기 때문에...

----

그 외에 느낀점은 JS 언어가 너무 별로다.

Java나 Kotlin 처럼 강타입 언어만 해서 그런가, 익명 함수나 덕타이핑을 쓰면 코드 구조를 파악하기 너무 어렵다. 
이게 타입이라도 있으면 익명 함수더라도 역할을 유추를 할 수 있을거 같은데, 그냥 인자로 받아서 실행해버리니까 이게 뭔지 모르겠음.
덕타이핑은 IDE에서 연동도 잘 안되는거 같고, 연관관계를 도저히 찾을수가 없었다.

핵심적인 함수인 `renderWrapper()`에서 사용되는 인자인 `component`가 `/view/*.js`로 구현된 컴포넌트 함수를 의미한다는 걸 이해하기까지 너무 오래 걸렸다.
